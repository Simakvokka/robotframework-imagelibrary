- Добавлена сущность buttons_panels. Указываются в конфиге. См. amazon.yaml для подробностей. Позволяют указать панель с кнопками одинакового размера по координатам.

v 0.2.3
- Добавлены ключевые слова Wait For Image To Stop, Wait For Template To Stop, Wait For Window To Stop. Все они подразумевают, что происходит ожидание, когда элемент появится на экране, затем ожидается, когда элемент остановится. Может быть полезно для всплывающих окон, анимированных кнопок и прочего.

v 0.2.2
- Добавлены анимированные кнопки - animating buttons. Для них можно указать несколько ключевых кадров, поиск происходит по любому из них. В остальном ведут себя так же, как обычные (static) кнопки.

v 0.2.1
- Добавлена возможность указывать несколько yaml файлов. При совпадении элементов (окон и т.п.) последующие файлы затирают предыдущие без сообщения об ошибке, so be carefull.

v 0.2.0
- Переделана инициализация - теперь все окна инициализируются в момент инициализации библиотеки. Ключевые слова Wait For Template можно вызывать сразу.
- Введено понятие "статические элементы" - это кнопки и изображения, определенные в секции "check". Предполагается, что они на экране всегда.
- Ключевое слово Init Window заменено на Check Window - производит все проверки, инциализирует статические элементы
- Также инициализация статических элементов происходит при первом обращении к одному из них (например, нажатие кнопки)
- Wait For Show/Hide также отвязаны от инициализации окна - таким образом, можно делать Wait For Show  some_window, а затем работать с ним
- Введено понятие "динамические кнопки", которые на экране не всегда. Перед нажатием происходит поиск кнопки на экране. Сделано ключевое слово Press Dynamic Button If On Screen

v 0.1.6
- Появился changelog.txt
- Улучшено информирование об ошибках. При неудаче любого метода slotbot сохраняется текущее состояние окна и сохраняются данные
- Во время игры стали доступны переменные уровня теста - ${game_mode} и ${game_state}
    - ${game_mode} - текущий режим игры, может принимать значения "normal" и "freegames". В планах режим "auto"
    - ${game_state} - текущее состояние игры, может принимать значения idle, rotate, win, gamble, bonus
    Данные значения можно использовать как для понимания, что происходит с тестом, так и для задания логики теста
- В конфиге желательно прописать game_name - имя игры (используется для создания скриншотов)
